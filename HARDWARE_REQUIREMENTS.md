# VisionForge 하드웨어 요구사항

> 기준일: 2026-07-12  
> 필수 기준 장비: MacBook Pro 16-inch, M1 Pro, 통합 메모리 16GB  
> 허용 학습 시간: 작업당 최대 72시간  
> 적용 범위: 로컬 학습, 실제 사진 일괄 추론, 결과 검수

## 1. 확정 전제

VisionForge의 현재 필수 실행 장비는 사용자가 바로 사용할 수 있는 `M1 Pro 16GB MacBook Pro`다. 별도 구매 예정 장비와 현재 Windows 개발 PC는 프로젝트의 최소 사양이나 합격 기준으로 사용하지 않는다.

- 지원한다고 표시한 작업 유형은 M1 Pro 16GB에서 학습·평가·추론·폴더 분류까지 완료되어야 한다.
- 전체 학습은 최대 72시간까지 허용한다.
- 하드웨어는 학습 속도와 사용 가능한 모델 크기에 영향을 주지만 상용 정확도를 결정하지 않는다.
- M1 Pro에서 학습한 모델도 실제 사진 고정 평가 세트의 품질 기준을 통과하면 상용 모델로 인정한다.
- 기준 장비에서 실행할 수 없는 작업 유형이나 모델은 임의로 축소 실행하지 않고 지원 불가 사유를 표시한다.
- 다른 장비는 필수 조건이 아니라 학습·추론 시간을 줄이는 선택형 가속 장비로만 취급한다.

## 2. 최소 사양

| 항목 | 필수 기준 |
|---|---|
| 장비 | MacBook Pro 16-inch, M1 Pro |
| CPU·GPU | M1 Pro 10-core CPU, 16-core GPU 기준 |
| 메모리 | 16GB 통합 메모리 |
| 운영체제 | macOS 14 이상 |
| 학습 가속 | PyTorch MPS, 미지원 연산은 검증된 CPU fallback |
| 추론 가속 | PyTorch MPS 우선, 미지원 연산은 CPU fallback |
| 저장 공간 | 시작 전 `max(100GB, 예상 학습 산출물 + 예상 분류 복사본 + 20%)` 확보 |
| 외장 저장소 | Thunderbolt NVMe 프로젝트·입출력 드라이브 지원 필수 |
| 학습 시간 | 기준 작업 72시간 이내 |

내장 SSD 용량을 최소 사양으로 고정하지 않는다. 프로그램이 실제 입력 파일, 생성 캐시, 모델 체크포인트와 결과 복사본의 예상 크기를 계산하고 내장 또는 외장 SSD의 여유 공간을 검사한다.

## 3. M1 Pro 16GB 학습 전략

### 3.1 모델과 학습

- 처음부터 대형 모델을 학습하지 않고 재배포 가능한 사전 학습 경량·중형 비전 모델을 미세조정한다.
- backbone 동결 학습 후 필요한 계층만 단계적으로 해제한다.
- 기본 batch는 1~2로 제한하고 gradient accumulation으로 유효 batch를 구성한다.
- 학습 입력은 기본 짧은 변 640px·긴 변 최대 960px로 사용하고, 작은 대상은 원본 고해상도 crop 또는 타일 데이터로 보완한다.
- 혼합 정밀도는 MPS 결과 일치와 안정성 시험을 통과한 모델에서만 사용한다.
- 여러 모델이 필요한 파이프라인은 동시에 메모리에 올리지 않고 순차적으로 학습한다.
- 실제 검증 점수가 개선되지 않으면 조기 종료하고 가장 좋은 체크포인트를 보존한다.

### 3.2 메모리와 저장량

- 이미지 전체 세트를 메모리에 올리지 않고 한 장 또는 제한된 prefetch만 디코딩한다.
- 학습 변형 이미지는 기본적으로 파일로 누적하지 않고 메모리에서 특징·텐서를 만든 뒤 해제한다.
- 원본 고해상도 사진을 통째로 큰 텐서로 유지하지 않고 축소 탐지와 필요한 crop·타일 처리를 조합한다.
- DataLoader worker와 prefetch 수는 가용 메모리 측정 결과에 따라 0~2 범위에서 제한한다.
- 운영체제 메모리 압박이 발생하기 전에 batch·worker·cache를 줄이고, 품질에 영향을 주는 해상도 변경은 사용자에게 표시한다.

### 3.3 장시간 안정성

- epoch 종료 또는 최대 30분 간격으로 재개 가능한 체크포인트를 저장한다.
- 프로그램 종료, 절전, 전원 분리와 프로세스 오류 후 마지막 정상 체크포인트부터 재개한다.
- 24시간 이상 시험에서 메모리 사용량이 지속 증가하지 않아야 한다.
- 전원 어댑터 연결과 절전 방지 상태를 학습 전에 확인하고 경고한다.
- 예상 완료 시간, 현재 epoch, 최고 검증 점수와 마지막 체크포인트 시각을 표시한다.

## 4. 기준 작업과 합격 조건

M1 Pro 16GB 하드웨어 수용 시험에는 다음 기준 작업을 사용한다.

- 사전 학습 경량 객체 탐지 모델 미세조정
- 학습 입력 최대 변 640px
- 라벨이 있는 유효 샘플 10,000개, 최대 50 epoch
- batch 1~2와 gradient accumulation
- 학습에 사용하지 않은 실제 사진 평가 세트
- 전체 학습 72시간 이내

합격 조건은 다음과 같다.

1. 메모리 부족으로 종료되지 않는다.
2. 중단 후 마지막 정상 체크포인트부터 재개된다.
3. 같은 체크포인트의 MPS·CPU 추론 결과가 허용 오차 안에서 일치한다.
4. 실제 사진 품질 기준을 통과한 모델만 자동 분류용으로 활성화된다.
5. 기준 작업보다 무거운 TaskSpec은 예상 시간과 메모리를 사전 계산해 지원 여부를 명확히 표시한다.

이 기준의 샘플 수는 모든 실제 프로젝트의 데이터 최소 수량이 아니라 장비와 백엔드의 반복 가능한 수용 시험 부하다.

## 5. 실행 프로필

| 프로필 | 조건 | 허용 범위 |
|---|---|---|
| `APPLE_M1_16_BASELINE` | M1 Pro, 통합 메모리 16GB | 지원 작업의 전체 학습·평가·추론, 최대 72시간 |
| `APPLE_HIGH_MEMORY` | Apple Silicon, 통합 메모리 24GB 이상 | 더 큰 batch·cache·모델 후보 허용 |
| `CUDA_ACCELERATED` | 검증된 NVIDIA CUDA 장비 | 동일 모델의 선택형 고속 학습·추론 |
| `CPU_FALLBACK` | GPU 가속 불가 | 프로젝트 관리와 지원 모델 추론, 학습은 별도 호환성 판정 |

프로필은 편의와 속도 설정에만 사용한다. `APPLE_M1_16_BASELINE`이라는 이유로 상용 학습이나 품질 통과를 차단해서는 안 된다.

## 6. 프로그램 자동 판정

프로그램은 실행 시 다음을 검사한다.

1. macOS 버전, Apple Silicon 종류와 통합 메모리
2. MPS·CPU 실행 공급자와 향후 추가되는 변환 런타임
3. 현재 가용 메모리, 메모리 압박과 swap 사용량
4. 프로젝트 및 출력 드라이브의 여유 공간과 예상 산출물
5. 고정 샘플의 학습·추론 처리량, 최대 메모리와 fallback 발생 여부

판정 결과에 따라 모델 후보, 입력 해상도, batch, gradient accumulation, worker, prefetch와 정밀도를 자동 설정한다. 기준 장비에서 예상 학습 시간이 72시간을 넘으면 더 작은 검증 모델 또는 crop·타일 전략을 선택하고 변경 이유를 표시한다. 품질 기준을 희생하는 자동 축소는 금지한다.

## 7. macOS 검증 절차

현재 저장소의 Windows 설치 앱은 M1 Pro 실행 증거가 아니다. 최종 지원 판정에는 실제 M1 Pro 16GB 장비에서 다음 검증이 필요하다.

1. arm64 엔진과 Tauri 앱 빌드
2. 설치·실행·프로젝트 복원과 외장 SSD 경로 검증
3. PyTorch MPS 학습 및 체크포인트 재개 시험
4. MPS·CPU 추론 결과 일치 시험
5. 실제 사진 일괄 처리와 원본 보존 폴더 라우팅 시험
6. 24시간 이상 메모리·절전·발열 안정성 시험
7. 기준 작업 예상 시간이 72시간 이내인지 확인

## 8. 선택형 가속 장비 원칙

추가 장비의 가격이나 사양은 VisionForge의 최소 요구사항에 포함하지 않는다. 향후 CUDA 장비나 다른 Apple Silicon 장비를 사용하더라도 동일 TaskSpec, 데이터셋, 평가 세트와 품질 게이트를 사용하며, 모델 패키지는 M1 Pro에서 다시 추론 검증을 통과해야 한다.
